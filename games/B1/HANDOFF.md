## 2. 게임 규칙 (확정)
- 대상 중2 · **2인 1조** · 한 장소 QR 1개(둘이 함께 풂, 협력 분할 없음)
- **N문항 주도**: 출제한 문항 수 N → station N개 자동 생성. 우승조건·라우팅·QR·HQ 전부 N에서 파생.
- **N개 중 need개(기본 N-3) 먼저 풀고 본부 복귀 = 우승.** (더미는 N=8, need=5)
- 문제: **단답형**, 고난도·시사상식 포함. **LaTeX 수식 지원.**
- **사진 유추 내비**: 앱이 다음 장소를 이름이 아니라 **사진 단서**로 제시 → 학생이 유추해 이동.
- **힌트 총량 예산**: 인솔교사 없음 → 앱 힌트, 단 조 전체 사용 개수 cap.
- **HQ 실시간 추적 유지.** 조 개수 유동(결석 대비 10~13조 무방).
- 참관교사용 안내문 별도(미작성). 균형 조편성(미구현).

## 3. 이동 모델 (안정성 핵심 — 사용자 1순위 = 안정성)
- **② 실시간 혼잡도 동적 배정**(채택): 풀 때마다 다른 조의 `currentTarget`을 읽어 **가장 한산한 미방문 station**을 배정 → 그 사진을 단서로. 동률은 조별 루트순(결정적, 무작위 없음)으로 분산.
- **① 결정적 루트 폴백**: 오프라인/실패 시 루트순 첫 미방문으로 자동 폴백 → **게임 안 멈춤.**
- **오프라인-우선**: 핵심 플레이(스캔→문제→채점→이동)는 전부 로컬. 네트워크는 보정일 뿐.
- WiFi 출렁여도(실내외 왕복) **재접속 시 HQ 추적 재개**면 OK라고 합의됨.

## 4. 파일 지도

## 7. 주의/미해결
- **Firebase**: A 프로젝트(`school-explorer-8e687`) 재사용, B는 `sessions/B1`로 분리(A와 충돌 없음). DB 규칙이 열려 있어야 ②/HQ 동작 — 잠겼으면 오프라인으로 안전 격하. **공개 DB 보안은 점검 필요**(A의 기존 이슈).
- **KaTeX는 현재 CDN** — 오프라인 위해 선캐시/번들 필요(오프라인 강화 단계).
- **실제 문제 미입력**(전부 더미). 출제.html로 채우면 자동 반영.
- **매직워드 프로토콜**: 작업 시작 전 컨펌, magic word(고/시작/파이어/go/start) 받고 실행.
