## 1. 게임 규칙 (어떤 게임인가)

- **대상:** 중2 · **2인 1조** · 크롬북 1대를 둘이 함께 사용
- **방식:** 학교 곳곳에 붙은 QR을 찾아 스캔 → 문제 풀기 → 다음 장소로
- **우승 조건:** 전체 문제(예: 16개) 중 **정해진 개수(예: 12개)**를 먼저 풀고 본부 복귀
- **다음 장소 안내:** 장소 이름이 아니라 **사진(또는 장소명)** 으로 단서 제시 → 학생이 유추해 이동
- **혼잡 분산:** 조마다 다른 순서로 이동하도록 자동 배정(한 곳에 안 몰림)
- **힌트:** 조 전체가 쓸 수 있는 힌트 개수에 상한(예: 6개)
- **수식 지원:** 문제에 LaTeX 수식($\triangle ABC$ 등) 사용 가능

현재 게임세트 **B1 = "중2 도형 단원평가 탐험 레이스"** (16문제 / 우승 12개 / 힌트 6 / 12개 조)

## 3. 게임 준비부터 운영까지 — 전체 순서

```

[배정]  ④ 개발자 페이지에서 게임 배정 → QR·대시보드 링크 발급
```

## 4. ① 문제 만들기 — 출제 빌더 (출제.html)

개발자 페이지 → **🛠 준비 탭 → 출제 빌더** 로 엽니다. (`출제.html?set=B1`)

### 화면 구성
- **게임 설정:** 제목, 우승 조건(need), 힌트 예산, 조 목록
- **문제 카드(station):** 문제마다 하나씩
  - **장소 표기:** 정답지·대시보드용 라벨 (예: "도서관 입구")
  - **장소 단서 사진(cluePhoto):** 학생에게 보여줄 "여기로 가라" 사진
  - **문제(question):** 단답형. `$수식$` 가능
  - **문제 이미지(questionImage):** 도형 그림 등
  - **힌트:** 여러 개 추가 가능
  - **정답(answer):** 여러 개 허용(쉼표 구분)

### 저장 방법 (중요)
- **Chrome 권장.** "📂 불러오기"로 폴더를 연결하면, "💾 저장"이 `sets/B1/stations.json`과 `images/`에 **직접 저장**됩니다.
- 폴더 연결을 안 하면 → JSON 파일이 **다운로드**됩니다(수동으로 레포에 넣어야 함).
- ⚠️ **저장만으로는 배포 안 됨.** 실제 학생에게 반영하려면 **git commit & push**가 필요합니다(개발자 작업).

### 이미지 경로 규칙
- 상대경로(`images/Q1.jpg`)로 적으면 자동으로 `sets/B1/images/Q1.jpg`에서 찾습니다.
- 외부 URL(`https://...`)은 그대로 씁니다.

## 5. ② QR 출력·부착 — QR 인쇄 (qrcodes.html)

개발자 페이지 → **🛠 준비 탭 → QR 인쇄** 로 엽니다.

- 각 station(장소)마다 **QR코드 1개**가 생성됩니다.
- QR 내용 = `play.html?s=장소번호` (학생이 스캔하면 그 문제로 진입)
- **게임 주소** 입력칸에 배포 주소를 넣으면 그 주소 기준으로 QR이 만들어집니다.
- 인쇄해서 **각 장소에 부착**합니다. (장소 단서 사진과 실제 장소가 맞아야 함)

## 6. ③ 반 명단 등록 — 개발자 페이지 (👥 반 명단 탭)

로그인 모드에서 학생이 **짝 이름을 명단에서 고르도록** 하려면 명단을 등록합니다.
(등록 안 하면 짝 이름을 자유 입력 — 오타·가명 위험)

### 양식: `반, 번호, 이름` (첫 줄은 머리글)
```
반,번호,이름
2반,1,김민준
2반,2,이서연
```
- 탭에서 **"샘플 CSV 받기"** 로 양식을 받을 수 있습니다.

### 등록 방법 3가지 (아무거나)
1. **CSV/엑셀 파일 업로드** (.csv, .xlsx)
2. **CSV 직접 붙여넣기** (텍스트 칸에 붙여넣고 "등록")
3. **구글시트 공개 링크** — 시트를 "링크가 있는 모든 사용자 보기"로 공개 후 주소 입력
   - ⚠️ 공개 시트에 학생 실명을 두면 노출 위험. 업로드/붙여넣기 권장.

→ 등록하면 반별로 저장되고, 배정 시 자동 연결됩니다. (이 PC에만 저장 — 공개 안 됨)

### 즉석 명단 (동아리 등 등록 안 된 그룹)
- 배정 탭에서 로그인 모드 선택 시, 등록 안 된 반이면 **"즉석 명단 업로드"** 칸이 나타납니다.
- 같은 양식으로 그 자리에서 올리면 이번 배정에만 적용됩니다.

## 7. ④ 게임 배정 — 개발자 페이지 (🎯 새 배정 탭)

게임을 실제로 "켜는" 단계입니다.

1. **게임세트** 선택 (현재 B1)
2. **대상(반)** 입력 (예: 2반) — 등록된 반은 자동완성으로 뜸
3. **회차** 입력 (같은 반에 또 돌리면 2, 3…)
4. **모드** 선택 (로그인/비로그인/테스트)
5. **[🚀 이 설정으로 게임 만들기]** 클릭

→ **배정 이력 탭**으로 이동하며, 방금 만든 게임이 카드로 뜹니다:
- **학생용 QR/링크** (이걸 학생에게 보여주거나 배포)
- **대시보드 링크** (로그인 모드만 — 운영용)
- 정보 칩: `📚 B1 · 👥 2반 · 🔁 1회차 · 🔑 로그인`

### "1반 끝 → 2반 시작"
- 1반 게임을 종료·보관하고, **새 배정**으로 2반을 만들면 됩니다.
- 새 반 = 새 게임(다른 방). 1반 데이터와 안 섞입니다. **문제·QR은 재사용**(다시 안 만듦).

### 배정 이력 (📋 탭)
- 배정한 게임이 게임세트별로 쌓입니다. 새로고침해도 유지(이 PC).
- 각 카드: QR · 학생링크 · 대시보드 · **🏁 종료·보관** · 삭제
- 끝난 게임은 "종료·보관" → "🗄 보관됨" 표시(되살리기 가능).

## 8. ⑤⑥ 운영 — 대시보드 (hq.html)

배정 카드의 **대시보드 링크**(로그인 모드)로 엽니다. 등록 교사 로그인 필요.

### 화면 구성
- **🏁 달성 순서:** 우승 조건 달성한 조의 순위(메달)
- **📡 실시간 조별 진행:** 각 조의 현재 위치·진행률·마지막 업데이트 시각
  - 조마다 **🛠 강제 이동** 버튼(특정 장소로 강제 배정 — 응급용)
- **📊 장소 혼잡도:** 지금 어느 장소로 몇 조가 향하는지(🔴 몰림 경고)
- **📚 커닝지(접힘):** 전체 문제·정답 (펼치기 — 학생 근처에서 펼치지 말 것)

### 운영 버튼 (상단)
- **📢 전체 공지:** 모든 학생 단말기에 메시지 전송 (빠른 문구: 5분 남음/본부 소집/잠깐 멈춰)
- **🏁 게임 종료·소집:** 게임 종료 + 결과 저장(아래 9번) + 학생 화면 마감
- **♻️ 진행 초기화:** 모든 조 진행 삭제(다시 시작용 — 주의)

## 9. ⑦⑧ 게임 종료 & 결과 채점

### 게임 종료 (대시보드 → 🏁 게임 종료·소집)
누르면 순서대로:
1. 모든 학생 화면에 **종료 안내**가 떠 진행이 멈춤
2. **현재까지의 결과가 영구 저장**됨 (완주 못 했어도 클리어 수 기록)
3. **구글시트에 자동 기록**

⚠️ **반드시 "게임 종료"를 눌러야 결과가 저장·기록됩니다.** 안 누르면 기록 안 됨.

### 결과가 어디에 저장되나
- **Firebase(영구 보관):** 초기화·덮어쓰기와 무관하게 보존되는 안전망
- **구글 스프레드시트(자동 기록):** 채점용. 시트에 자동으로 행이 추가됨
  - 시트 헤더: `종료시각 · 세션 · 게임세트 · 반 · 회차 · 모드 · 조 · 내이름 · 짝이름 · 이메일 · 클리어수 · 전체 · 우승조건 · 완주 · 완주시각`
  - 같은 게임을 또 종료하면 그 게임 행을 **덮어씀**(중복 안 쌓임)

## 10. 학생 입장에서 (play.html)

학생은 **배정 QR/링크**로만 들어옵니다.
1. (로그인 모드) 학교 구글 계정 로그인 — **한 명만**
2. 우리 조 선택 + 내 이름 + **짝 이름(명단에서 선택)**
3. "다음 장소" 사진(또는 이름)을 보고 이동 → QR 스캔 → 문제 풀기
4. 우승 조건만큼 풀고 본부 복귀 → 완주

- QR 없이 그냥 play.html을 열면 **"QR로 입장하세요"** 안내가 뜸(정상).
- 카메라(QR 스캔)는 **HTTPS(배포본)에서만** 동작.

## 12. 새 게임세트(B2, B3…) 추가하기

문제 묶음을 새로 만들려면 (개발자 작업):
1. `games/B1/sets/B1` 폴더를 복사해 `sets/B2`로
2. 출제 빌더를 `출제.html?set=B2`로 열어 문제 편집
3. `sets/index.json`에 한 줄 추가: `{ "id": "B2", "title": "제목", "count": 문제수, "need": 우승조건 }`
4. **git commit & push** — 이래야 배포됨
5. 개발자 페이지 새로고침 → 드롭다운에 B2가 뜸
- 자세한 건 `games/B1/sets/README.md` 참고.

## 14. 문제 해결 (자주 묻는 것)

| 증상 | 원인·해결 |
|---|---|
| 로그인이 깜빡하고 실패 | 로컬(127.0.0.1)에선 안 됨. **배포본(HTTPS)**에서 등록 교사 계정으로. |
| 이미지가 안 보임 | 경로가 `sets/{세트}/images/`인지 확인. 출제 빌더 미리보기는 새로고침. |
| 대시보드가 비어있음 | 로그인 모드인지 확인. 비로그인/테스트는 추적 안 함. |
| 결과가 시트에 안 들어감 | "게임 종료"를 눌렀는지 / 등록 교사로 로그인했는지 확인. |
| 학생이 "준비 중"만 봄 | 그 게임세트가 아직 push 안 됨(배포 필요). |
